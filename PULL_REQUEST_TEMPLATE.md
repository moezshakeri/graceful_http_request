## Before Submitting

- [ ] Ensure all tests pass: `flutter test`
- [ ] Ensure no analyzer issues: `flutter analyze`
- [ ] Add tests for new features or bug fixes
- [ ] Update documentation if needed
- [ ] Follow commit message conventions

## PR Checklist

### Code Quality
- [ ] Code follows existing style
- [ ] Code is self-documenting
- [ ] Public APIs have DartDoc comments
- [ ] No TODOs or FIXMEs left in code

### Testing
- [ ] All tests pass locally
- [ ] New tests added for changes
- [ ] No flaky tests introduced
- [ ] Coverage maintained or improved

### Documentation
- [ ] README updated if needed
- [ ] API docs updated for public changes
- [ ] CHANGELOG.md updated with version entry

### Platform Testing
- [ ] Tested on Android
- [ ] Tested on iOS
- [ ] Tested on Web
- [ ] Tested on relevant desktop platforms

## Review Process

1. Automated checks must pass
2. At least one maintainer must review
3. Address all review comments
4. Get approval from maintainers
5. Squash and merge as needed

## Questions?

Feel free to ask questions in your PR if anything is unclear!
